MySQL EXPLAIN 解读

2022/07/05 posted in  MySQL
Tags: 

EXPLAIN作为MySQL的性能分析神器，可用来分析SQL的执行计划

EXPLAIN 示例

EXPLAIN SELECT
	c.PK 
FROM
	consignments c
	JOIN pointofservice p ON p.PK = c.p_pointofservice
	JOIN orders o ON o.PK = c.p_order 
WHERE
	(
	c.p_consignmentdoctype IN ( 8796118450267, 8796118351963, 8796118384731 ) 
	AND c.p_status = 8796104753243 
	AND c.p_accountdocstatus = 8796093087835 
	AND (
	( o.p_orderbstype = 8796854648923 AND p.p_bstype IN ( 'DS', 'JM', 'JR' ) ) 
	OR ( o.p_orderbstype = 8796854616155 AND p.p_bstype IN ( 'MD', 'RD', 'DS' ) ) 
	OR ( o.p_orderbstype = 8796854681691 AND p.p_bstype = 'DS' ) 
	OR ( o.p_orderbstype = 8796854583387 AND p.p_bstype = 'JM' ) 
	) 
	) 
	AND ( ( c.TypePkString = 8796107407442 AND p.TypePkString = 8796095250514 AND o.TypePkString = 8796094136402 ) ) 
	LIMIT 0,
	1000;

字段介绍

字段 描述
id 表示查询中的每个操作的唯一标识符，如果explain的结果包括多个id值，则数字越大越先执行；而对于相同id的行，则表示从上往下依次执行。
select_type 表示查询类型，常见的有SIMPLE、PRIMARY、SUBQUERY、DERIVED、UNION等。
table 表示表的名称或者表的别名。
partitions 表示访问的分区。
type 表示访问类型，包括：system、const、eq_ref、ref、fulltext、ref_or_null、index_merge、unique_subquery、index_subquery、range、index、ALL等。
possible_keys 表示查询可能使用的索引。
key 表示实际使用的索引。
key_len 表示索引字段的长度。
ref 表示使用索引的哪个列与表的哪个列进行比较。
rows 表示MySQL估计需要扫描的行数。
filtered 表示查询结果集的行数与扫描的行数之比。
Extra 表示执行查询时的额外信息，包括Using filesort、Using temporary、Using index等。

select_type介绍

select_type 描述 示例
SIMPLE 简单的SELECT查询，不使用UNION或子查询等。 SELECT * FROM table WHERE column = 123
PRIMARY 最外层的查询，即包含所有的子查询的查询。 SELECT * FROM table WHERE column IN (SELECT column FROM ...)
UNION UNION中的第二个或后面的SELECT查询。 SELECT column FROM table1 UNION SELECT column FROM table2
DEPENDENT UNION UNION中的第二个或后面的SELECT查询，依赖于外部查询的结果。 SELECT * FROM table1 WHERE column = (SELECT column FROM table2 WHERE ...)
UNION RESULT UNION的结果集。 (SELECT column FROM table1) UNION (SELECT column FROM table2)
SUBQUERY 子查询中的第一个SELECT查询，该查询在外部查询之前执行。 SELECT * FROM table WHERE column = (SELECT column FROM ...)
DEPENDENT SUBQUERY 子查询中的第一个SELECT查询，该查询依赖于外部查询的结果。 SELECT * FROM table1 WHERE column = (SELECT column FROM table2 WHERE ...)
DERIVED 派生表的SELECT查询，包含子查询中的第一个SELECT查询，该查询在外部查询之前执行。 SELECT * FROM (SELECT column FROM table1) AS t
MATERIALIZED 派生表的SELECT查询，包含子查询中的第一个SELECT查询，该查询在外部查询之前执行，但结果会被缓存起来以便后续查询使用。 WITH t AS (SELECT column FROM table1) SELECT * FROM t

type

需要注意的是，排序是根据查询效率从高到低排序的，并不是绝对的顺序。同时，不同类型的查询在不同的场景下具有不同的效率和优化策略，具体的查询优化需要根据实际情况进行调整。同时，索引示例和SQL示例仅用于说明，实际查询语句和索引结构可能更加复杂。

type 排序 描述 索引示例 SQL示例
all 8 全表扫描，性能最差。 NULL SELECT * FROM table
index 7 全表扫描，但只扫描索引。 INDEX SELECT column FROM table WHERE column LIKE 'value%'
range 6 使用索引范围匹配查询，通常使用BETWEEN、<、>、<=、>=等操作符。 INDEX SELECT * FROM table WHERE column BETWEEN 1 AND 10
index_subquery 5 在子查询中使用普通索引进行匹配。 INDEX SELECT * FROM table WHERE column IN (SELECT column FROM ...)
unique_subquery 4 在子查询中使用唯一索引进行匹配。 PRIMARY SELECT * FROM table WHERE column = (SELECT column FROM ...)
index_merge 3 使用多个索引进行匹配，然后将结果合并。 INDEX1, INDEX2 SELECT * FROM table WHERE column1 = 'value' OR column2 = 'value'
ref_or_null 2 和ref类似，但包含了NULL值的行。 INDEX SELECT * FROM table WHERE column = 'value' OR column IS NULL
ref 1 使用普通索引查找匹配的查询，通常用于一个表中查找一些数据。 INDEX SELECT * FROM table WHERE column = 'value'
eq_ref 0 使用唯一索引查找匹配的查询，通常是多表连接中一个表的主键或唯一索引作为另一个表的外键。 PRIMARY, UNIQUE SELECT * FROM table1 JOIN table2 ON table1.id = table2.id
const -1 使用常数值匹配的查询，例如使用主键或唯一索引进行精确查找。 PRIMARY SELECT * FROM table WHERE id = 123
